介绍

本项目示例用来快速创建基于webpack + es6开发环境，以支持开发以下类型项目：

  • SPA 单页应用
  • MPA 多页面应用

由于增加了对jsx的编译识别，所以可以在项目中随意使用react全家桶。当然，使用jquery也是没有任何问题的。

上手使用

开始开发前，要了解以下几个基本情况：

  • 页面入口文件放置在 public 目录下，js入口文件放在 app 目录下（入口文件需直接放置与该目录下，如果嵌套文件夹放置，则不会被当作入口文件）。 其中html文件名与其要调用的js文件名保持一致。例如：piblic/index.html 对应于 app/index.js
  • package.json文件
    • vendor：该字段用来指定需要单独合并打包放置的模块，本示例默认打包了 jquery、react、react-dom和公共css模块。这在多页面应用中会很有用，可以抽取公共的模块单独打包，避免单一文件过大。
    • noRewrite: 本地server服务器是够重写所有访问路径至默认入口，默认是true。如果开发的是单页应用，则可以关闭该选项。
    • proxy: 代理服务器地址，如果指定了该参数，本地server可以代理所有接口请求到目标服务器，可以很方便的避开跨域限制。
  • 常用命令：
    • npm start 用来在开发时使用，运营本命令后，会在本地启动一个node server服务，同时用浏览器打开。开发时保存文件时可以实时刷新浏览器（支持hot reload）。
    • npm run build:dev build测试包，打包后的文件未经过压缩合并，包含了map信息，方便测试环境查错。
    • npm run build build线上包，打包后的文件经过压缩合并，保证最小size。
    • npm run pack build线上包并同时上传到cdn。需要上线时运行该命令。

由于增加爱了对jsx的编译识别，所以可以在项目中随意使用react全家桶。当然，使用jquery也是没有任何问题的。

使用示例

  • js里引用不同位置模块

    import API from 'utils/API'; //引用app/utils下模块, 也可以通过 import API from 'app/utils/API';
import FeedBack from 'compoennts/FeedBack'; //引用app/components下模块
import APP from 'modules/APP'; //引用app/modules下模块
import favicon from 'public/favicon.ico'; //引用其它位置模块

  • html模版中引用图片等静态资源

    <img src="/images/cat.jpg" width="200" />
<img src="../static/images/cat.jpg" width="200" />

    同理适用于link、audio、video等标签，引用音视频资源

  • react组件里引用静态资源（下面demo为react组件）

    import React, { Component } from 'react';
import img from '../../../static/images/cat.jpg'; //或者
// import img from 'static/images/cat.jpg';

class ImgDemo extends Component {
    render() {
        return (
            <img src={img} width="200" />
        );
    }
}
  • 多页面应用，你可以点击此处查看另一页面
${require('./layout/footer.html')}
